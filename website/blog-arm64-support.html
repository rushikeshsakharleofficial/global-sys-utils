v2.1.15 — ARM64 Package Support + Retry Jitter

May 2026

ARM64 / aarch64 packages

v2.1.15 ships the first official ARM64 packages for global-sys-utils. This adds support for:

  • Apple Silicon Macs running Linux VMs (UTM, Parallels, Lima)
  • AWS Graviton2/Graviton3 EC2 instances
  • Ampere Altra and other cloud ARM instances
  • Raspberry Pi 4/5 running 64-bit Raspberry Pi OS or Ubuntu
# Debian / Ubuntu arm64 wget https://github.com/rushikeshsakharleofficial/global-sys-utils/releases/download/v2.1.15/global-logrotate_2.1.15-1_arm64.deb sudo dpkg -i global-logrotate_2.1.15-1_arm64.deb sudo systemctl enable --now global-sys-utils.service # RHEL / Fedora aarch64 wget https://github.com/rushikeshsakharleofficial/global-sys-utils/releases/download/v2.1.15/global-logrotate-2.1.15-1.aarch64.rpm sudo rpm -ivh global-logrotate-2.1.15-1.aarch64.rpm

The Go core binary is compiled with GOARCH=arm64. Python cloud modules run on the system Python 3.9+ — no architecture-specific Python packages are required.

Retry jitter for cloud uploads

In multi-node environments where several servers run global-sys-utils on the same schedule, a transient S3 or GCS error caused all nodes to retry simultaneously — amplifying load on the cloud endpoint and causing cascading failures.

v2.1.15 adds exponential backoff with random jitter to all cloud upload retries. Each retry waits base * 2^attempt + rand(0, jitter_ms) milliseconds, spreading retry storms across a configurable window.

# Configure retry behaviour in global.conf UPLOAD_MAX_RETRIES=5 UPLOAD_RETRY_BASE_MS=500 UPLOAD_RETRY_JITTER_MS=300 # Retry delays: ~500ms, ~1.3s, ~2.9s, ~6.1s, ~12.6s
