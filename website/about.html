About global-sys-utils

Why it exists

Standard logrotate was designed in a different era — local disks, no cloud, single-node setups. Modern infrastructure runs distributed services, generates gigabytes of logs per hour, and requires encrypted off-site storage to meet compliance requirements.

global-sys-utils fills that gap. It's a single binary that handles the full lifecycle: parallel compression → AES-256-GCM encryption → S3/GCS upload → disk-space guard → systemd scheduling.

The conf.d job system means each service team manages their own log policy independently, without coordinating through a central config file.

Technical foundation

Core rotation engine written in Go — race-detector-clean, statically linked.
Cloud provider modules in Python — AWS boto3 and GCP google-cloud-storage.
systemd service + timer units installed automatically by the .deb/.rpm post-install scripts.
AES-256-GCM encryption with credential auto-load from file or environment variable.
Atomic write pattern (write-to-tmp, rename) prevents partial archives on crash.
DEB packages for amd64 and arm64; RPM packages for x86_64 and aarch64.
