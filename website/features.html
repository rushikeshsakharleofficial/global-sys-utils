Feature deep dive

Every capability global-sys-utils ships — how each one works and why it matters for production.

Core

Parallel Rotation

Rotate N log files concurrently, sorted by file size descending so the largest files start first. Worker count is configurable via --parallel N.

OS-level concurrency — each file on its own goroutine
Sorted by size for optimal throughput utilisation
Default worker count matches logical CPU count
parallel rotation
$global-logrotate--parallel 8/var/log/app/
[worker-1] rotating api.log (12.4 MB)
[worker-2] rotating worker.log (8.1 MB)
[worker-3] rotating cron.log (2.3 MB)
✓ 3 files in 1.8s (8 workers)
Security

AES-256-GCM Encryption

Archives encrypted with AES-256 in GCM mode before any cloud upload. Passwords stored as SHA-256 hashes; never plaintext.

One-time setup: --pass-gen
Auto-loads from ~/.global-sys-utils/credentials.ini
Override via LOGROTATE_PASSWORD env var
# Generate and store password (one time) global-logrotate --pass-gen # All subsequent rotations auto-decrypt global-logrotate --encrypt /var/log/app.log # → app.log.2026-06-16.gz.enc # Or pass via environment LOGROTATE_PASSWORD=secret global-logrotate --encrypt
Cloud

AWS S3 + GCS Backup

Upload aged archives directly to S3 or Google Cloud Storage. Each conf.d job independently specifies its cloud provider.

S3: named-profile support, MD5 verification, retry with jitter
GCS: ADC credential resolution, project-level auth
Dry-run preview before any actual upload
Adaptive upload throttle based on live CPU/RAM
# /etc/global-sys-utils/global.conf.d/app.conf LOG_DIR=/var/log/app CLOUD_PROVIDER=aws S3_BUCKET=my-log-archives AWS_PROFILE=prod-backup RETENTION_DAYS=90 # Manual upload with dry-run global-logrotate --upload --dry-run
Reliability

Disk Pressure Guard

Two-level disk protection: emergency rotation when disk usage crosses a threshold; archive write guard that aborts if free space would run out.

DISK_CRITICAL_PERCENT — triggers emergency rotation
DISK_MIN_FREE_MB — minimum free space before archiving
Daemon polls on configurable interval
DISK_CRITICAL_PERCENT=90 DISK_MIN_FREE_MB=200 DISK_CHECK_INTERVAL=60 # When disk > 90% used: INFO disk pressure detected (91%), triggering emergency rotation INFO freed 4.2 GB in 3.1s
Scheduling

Daemon + systemd

Full-featured daemon with cron expressions, interval aliases, and systemd service integration installed automatically by the package.

Aliases: @daily, @hourly, 6h
Standard cron expressions: 0 2 * * *
Oneshot systemd timer for single-run scheduled jobs
# Start daemon on interval global-logrotate --daemon --schedule 6h # Or standard cron expression global-logrotate --daemon --schedule "0 2 * * *" # Or use systemd (installed by package) sudo systemctl enable --now global-sys-utils.service
