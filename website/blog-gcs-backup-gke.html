Setting Up GCS Backup on GKE

January 2026

Overview

Running global-sys-utils as a DaemonSet on GKE lets you rotate and archive logs from every node to Google Cloud Storage without maintaining per-node credentials. This guide uses GKE Workload Identity — the recommended approach that eliminates service account key files entirely.

Prerequisites

  • GKE cluster with Workload Identity enabled (--workload-pool=PROJECT_ID.svc.id.goog)
  • GCS bucket for log archives
  • global-sys-utils v2.1.15+ (arm64 support for Autopilot nodes)

1. Create the GCS bucket

gcloud storage buckets create gs://my-gke-logs \ --location=US \ --uniform-bucket-level-access \ --lifecycle-file=lifecycle.json
# lifecycle.json — auto-delete after 90 days { "lifecycle": { "rule": [{ "action": { "type": "Delete" }, "condition": { "age": 90 } }] } }

2. Create a Google Service Account

gcloud iam service-accounts create gsu-log-uploader \ --display-name="global-sys-utils log uploader" gcloud storage buckets add-iam-policy-binding gs://my-gke-logs \ --member="serviceAccount:gsu-log-uploader@PROJECT_ID.iam.gserviceaccount.com" \ --role="roles/storage.objectCreator"

3. Bind to Kubernetes Service Account

kubectl create serviceaccount gsu-sa --namespace kube-system gcloud iam service-accounts add-iam-policy-binding \ gsu-log-uploader@PROJECT_ID.iam.gserviceaccount.com \ --role="roles/iam.workloadIdentityUser" \ --member="serviceAccount:PROJECT_ID.svc.id.goog[kube-system/gsu-sa]" kubectl annotate serviceaccount gsu-sa \ --namespace kube-system \ iam.gke.io/gcp-service-account=gsu-log-uploader@PROJECT_ID.iam.gserviceaccount.com

4. Create the ConfigMap

kubectl apply -f - <<EOF apiVersion: v1 kind: ConfigMap metadata: name: gsu-config namespace: kube-system data: global.conf: | LOG_DIR=/var/log/pods CLOUD_PROVIDER=gcp GCS_BUCKET=my-gke-logs RETENTION_DAYS=7 ENCRYPT=true DISK_CRITICAL_PERCENT=85 DISK_MIN_FREE_MB=500 EOF

5. Deploy as a DaemonSet

kubectl apply -f - <<EOF apiVersion: apps/v1 kind: DaemonSet metadata: name: global-sys-utils namespace: kube-system spec: selector: matchLabels: app: global-sys-utils template: metadata: labels: app: global-sys-utils spec: serviceAccountName: gsu-sa containers: - name: gsu image: debian:bookworm-slim command: ["/usr/bin/global-logrotate", "--daemon", "--schedule", "6h"] volumeMounts: - name: config mountPath: /etc/global-sys-utils - name: varlog mountPath: /var/log securityContext: runAsUser: 0 volumes: - name: config configMap: name: gsu-config items: - key: global.conf path: global.conf - name: varlog hostPath: path: /var/log EOF

Verify uploads

# Check pod logs kubectl logs -n kube-system -l app=global-sys-utils --tail=50 # List uploaded archives gcloud storage ls gs://my-gke-logs/ --recursive | head -20

Notes

  • Workload Identity tokens are automatically refreshed — no credential rotation needed.
  • The LOGROTATE_PASSWORD env var can be injected from a Kubernetes Secret for encrypted archives.
  • On Autopilot clusters, use node taints to ensure the DaemonSet schedules correctly on spot nodes.
